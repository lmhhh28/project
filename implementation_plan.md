# 城市通信线路连通与布线系统 - 实现计划

本应用旨在实现一个简化的国家通信线路网规划软件，将城市抽象为二维欧几里得平面的顶点，通信线路抽象为线段（边）。本项目使用核心 Java (无外部依赖库) 来确保最大的跨平台通用性和易编译性。

## 1. 整体架构 (MVC模式)
- **Model (数据模型)**:
  - `City`: 属性包括 `id`, `name`, `x`, `y`, `description`。
  - `Edge`: 属性包括 `source`, `target`, `length` (由欧几里得距离 $L = \text{round}(\sqrt{\Delta x^2 + \Delta y^2})$ 给出)。
  - `GraphState`: 维护城市集合和边缘集合，以及邻接表形式的图结构。
- **View (图形化界面)**:
  - 基于 **Java Swing** (`JFrame`, `JPanel`)。
  - `GraphCanvas`: 用于实时绘制城市节点（圆圈/图标）和通信线路（直线）。
  - `ControlPanel`: 右侧/底部的操作控制台，包含按钮、表单（添加城市/线路）以及状态输出文本框。
- **Controller (业务逻辑与控制)**:
  - 绑定所有 UI 按钮的事件监听器，处理 Model 更新并触发 View 重绘。
  - `FileManager`: 负责使用自定义 TXT 格式或对象序列化将模型数据保存/读取到本地文件。
  - `AlgorithmEngine`: 封装所有的图算法，执行相关计算返回给 Controller，再由 View 进行高亮显示。

## 2. 核心功能及算法设计

### 2.1 基础信息管理与文件IO (Req 1-4)
- 通过表单输入城市及线路的增删改查。
- 坐标修改、信息删除时，相关的依赖关联线路同步删除。
- **持久化方案**: 采用简单易读的结构化 TXT 文件（或序列化）：
  - 城市列表段落：`ID,Name,X,Y,Description`
  - 线路列表段落：`SourceID,TargetID`

### 2.2 连通图判断及最小辅助线路生成 (Req 5)
- **连通性计算**: 采用 BFS 或并查集 (Union-Find) 统计图中的连通分量 (Connected Components)。
- **最小增补线路**: 若不连通，在不同的连通分量之间建立“虚拟完全边”，边权为几何距离。然后运行 **Kruskal 最小生成树算法**，找出连接所有连通分量且总长度最小的增补边集合。最终在 UI 上用虚线高亮标出这些推荐边。

### 2.3 单源最短路径 (Req 6)
- **算法**: **Dijkstra** 算法。
- **输出**: 计算指定源城市到所有其他连通城市的最短路径距离，并按照距离从小到大排序展示。在图上实现点击某个目标城市后高亮出最短完整路径功能。

### 2.4 遍历全图最小路径与TSP问题 (Req 7)
TSP (旅行商问题) 为 NP-Hard 问题。用户需要两类查询：
1. **最短哈密顿回路 (TSP)**: 从某城出发经过所有城并回到起点。
2. **最短哈密顿路径**: 从某城出发经过所有城但不需回起点。
- **实现策略**:
  - 对于小规模城市 ($N \le 12$): 使用 **带记忆化的深度优先搜索 (状态压缩DP / 回溯法)** 求解全局绝对最优解。
  - 对于大规模城市 ($N > 12$): 实现 **最近邻启发式算法 + 2-Opt 局部优化** 来获取高质量的近似解。

### 2.5 施泰纳最小树近似网络布线 (Req 8)
完全空白情况下的 N 点最短相连，允许增加辅助点（即 Steiner Minimum Tree 问题）。
- **实现策略**: 精确求解欧几里得施泰纳树极难，我们将采用一种常用于工程软件的近似物理启发算法：
  1. 首先求出这 N 个城市的**最小生成树 (MST)** 作为基准 (Base Approximator)。
  2. 迭代改进：对于生成树中相邻的任意三个顶点（夹角小于 120 度时），计算它们的**费马点 (Fermat-Torricelli Point)** 作为增加的辅助点，用以替换原有的两条边，从而缩小整体布线长度。
  3. 将最终优化后的网络及新增辅助点在画布上做特殊颜色标记。
