# 城市通信网络规划系统 - 用户使用手册

本手册将指引您如何使用《城市通信网络规划系统》进行各类城市布线数据操作与各算法模块的测试。

## 1. 软件使用准备与启动
在确保您的计算机已安装 JDK 8 或更高版本的情况下，进入项目的根目录，编译并运行主程序即可启动：
```bash
# 1. 编译代码
javac -d bin src/com/citygraph/**/*.java src/com/citygraph/*.java

# 2. 启动系统
java -cp bin com.citygraph.Main
```

启动后，您将看到左侧的**可视化地图画布**，右侧的**控制面板（表单、算法指令）**，以及一个独立的**系统日志输出窗口**。
程序具备以下视觉亮点：
- **HiDPI 全中文适配**：字体抗锯齿且适应高分屏显示。
- **直观的数学直角坐标系**：画布中心锚定为 `(0,0)` 原点，Y轴向上为正，完全对齐数学几何直觉，配以自带刻度的辅助网格线底图，让每一条连接线的欧几里得距离一目了然。

## 2. 数据格式与坐标文件样例 (.txt 格式)

系统支持通过“数据存取”面板的 **[加载文件]** 和 **[保存文件]** 按钮，一键导入或导出现有的城市网络拓扑数据。文件需要遵循以下结构，要求纯文本（例如 `data.txt`），无特殊的编码绑定：

### 格式约定：
- 必须有 `[CITIES]` 分组标识城市点云。每个城市占用一行，采用逗号分隔，字段顺序为：`ID,名称,X坐标,Y坐标,简介`。
- 必须有 `[EDGES]` 分组标识已经连通的线路。每条线路占用一行，采用逗号分隔，字段顺序为：`起点ID,终点ID,边的长度`（边的长度随意写，因为系统内部会自动根据坐标的欧几里得距离强行矫正此段线路的长度）。

### 标准坐标文件样例 (直接复制为 `map.txt` 即可使用):
```text
[CITIES]
1,北京,200,100,首都
2,上海,300,300,直辖市
3,广州,250,500,南方节点
4,成都,100,400,西南枢纽
5,武汉,220,380,九省通衢
6,西安,150,250,西北重镇
7,乌鲁木齐,30,80,西域风情
[EDGES]
1,5,0
1,6,0
2,5,0
3,5,0
4,6,0
4,5,0
```

## 3. 基础功能使用指南

### 3.1 城市节点管理
此区域控制画布上的蓝色圆点（城市）。
- **添加城市**：在表单填入自定义的 `ID (整数)`、`名称`、`坐标 (如 100, 200)` 等，点击 **[添加城市]**。画布中心会出现新城市点。
- **修改城市**：如果想移动已有城市或改名，可以填写原本的城市 ID，填入新坐标等参数后点击 **[修改城市]**。与其相连的所有通信线长度均会自动热更新！
- **快捷点选**：也可直接在左侧画布上**用鼠标左键点击任意蓝色城市点**，其资料会一键自动回填到右侧表单中。
- **删除城市**：填入待删的 ID 点击即可，此城市及其关联的所有线路将被一同清除。

### 3.2 通信线路管理
此区域控制城市之间的黑色连线（实际通信线路）。
- **添加线路**：输入已存在的两个城市 ID，点击 **[添加线路]**，画布将即时产生一条通信黑线。
- **删除线路**：输入已连接的两个城市 ID，点击 **[删除线路]**，黑线即刻阻断销毁。

### 3.3 画布清空管理
若需要彻底打扫测试场景，可以点击数据存取区块最右侧的 **[清空画布]**，内存及界面将被恢复为空白状态。

## 4. 算法功能操作演示 (以 Q5-Q8 题号划分)

> 💡 **小贴士**：所有的算法由于生成了新的（彩色）辅助线，当您想要看清原本网络面貌时，随时点击面板最下方的 **[清除高亮与增补的辅助虚线]** 即可将画布重置回干净的黑蓝初态。

### Q5：检查连通性 / 生成增补网 (Kruskal)
- **目标**：测试当前国家蓝点图是否在物理线路上是100%连通的。
- **使用方法**：直接点击该按钮。
- **效果**：系统日志会打印出连通情况。若*不连通*，则画布会自动计算并用**蓝色虚线**将各个孤岛块以最小边连起来，确保全盘最经济地畅通无阻。

### Q6：计算单源最短路径 (Dijkstra)
- **目标**：从指定的某一个城市散发，求前往各地的最短途径。
- **使用方法**：先在 `目标城市 ID` 框中填入起点的 ID，然后点击 Q6 按钮。
- **效果**：生成一条源点通往全部相连节点城市的（标红加粗）树状最短路径拓扑。并且右下方日志区域，将按**自近到远的顺序**输出其余城市具体绕行的路线名及步长。

### Q7：旅行商最短路线 / 巡回问题 (TSP)
- **目标**：从某个源点出发，寻遍整个图内的蓝点一圈消耗的极端最低距离。
- **使用方法**：填入 `目标城市 ID` 作为跑分首发点，根据需求选择点击：
   - **[TSP - 不回起点]** (要求哈密顿路径：最后落脚在异地即可)
   - **[TSP - 回到起点]** (要求哈密顿回路：必须返回目标 ID 再闭环)
- **效果**：日志区将会输出 TSP 得出的详细城市经停顺序以及最终里程长。（内部拥有自适应模式：超12个城市会自动降格为最近邻启发的次优解保速度，小地图则自动使用回溯精算）。

### Q8：生成施泰纳最小树 (Steiner Tree)
- **目标**：抛开现有的所有路线，在“一块全新的空地”中仅仅保留这些指定的蓝点城市，可以借助添加辅助路由服务器点（计算出几何最佳交汇），重塑一张更经济的互联极简网。
- **使用方法**：此算法在“平底起高楼”上进行，因此直接点击 Q8 按钮即可触发**毁灭性的重构**。*(系统已内置安全备份，完成后可随时点击“清除高亮”完整恢复原貌！)*
- **效果**：系统会清空已有旧边，找出图节点三叉重心的费马点，在画布上画下新增的**橙色小方格**（虚拟辅助点），并自动为您布入**橙色实体线**，此时网络处于理论局部绝对极小连通带状态。

## 5. 性能与基准测试 (Benchmark)

本系统内置了一个专业的前后端分离式算法压测套件 `Benchmark.java`，为您提供生成 50、100、200 乃至最高 **500个城市节点** 网络场景的极速性能检测能力。

要启动压测模块，请在终端执行以下纯量级静默指令：
```bash
java -cp bin com.citygraph.test.Benchmark
```

控制台将会分级列印出 Kruskal 连通、Dijkstra 最短寻路、TSP 以及 Steiner 重建等四大算法组群在不同 N 规模下的**毫秒级时间花销**以及**堆内存波动记录**，同时该程序内部会强制执行一遍破坏性动作后验证结构完整性的压力巡检。
